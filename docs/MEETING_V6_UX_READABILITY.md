# 🏛️ ECC 團隊會議 — 首頁可讀性革命 (UX Refine)
**日期**：2026-02-28  
**議題**：在「暗黑美學」與「清晰閱讀」之間取得完美平衡

---

## 📋 問題診斷：美感掩蓋了實用性

### 🔴 用戶回饋
> 「有些顏色看不太清楚或者字體過小無法看清」

### 🔍 UX Reviewer 實地稽查
經過深度走訪首頁，我們發現了以下 3 個「視力殺手」區域：

1.  **灰階文字對比度不足**：
    *   許多說明文字使用 `text-zinc-500` 或 `text-zinc-600`，在純黑背景下，對於長輩或在陽光下的手機用戶來說幾乎是「保護色」。
2.  **關鍵副標題過小**：
    *   章節標籤、小標題的字體（如 `text-[10px]`）在行動端上顯得過於破碎，無法起到導航作用。
3.  **打字機效果背景干擾**：
    *   動態粒子背景雖然酷炫，但若文字不夠「亮」，粒子會干擾視線。

---

## 🎯 改造方案：高對比度、大字體、清晰層次

### 🧠 Planner 提案：文字分級與標準化

#### A. 顏色提亮計劃
*   **正文/說明文字**：從 `zinc-500/400` 全面提升至 `zinc-300` (更白一點)。
*   **關鍵強調文字**：全面提升至 `emerald-400` 或 `white`，並加粗。
*   **暗處標註**：最低限度使用 `zinc-400`，嚴禁使用 `zinc-600` 作為閱讀內容。

#### B. 字體「大」進化
*   **Hero Subtitle**：從 `text-lg` 升級至 `text-xl`，增加行高 (leading-relaxed)。
*   **Card Description**：確保不小於 `text-base` (16px)。
*   **Metadata (時間/篇數)**：從 `text-[10px]` 升級至 `text-xs` (12px)，並改為全大寫加粗。

#### C. 區塊對比增強
*   **Sponsor Card**：文字改為全白/亮綠，確保在漸層背景上清晰可見。
*   **Before/After**：加強文字粗度，讓對比一眼就能被掃描到。

---

## 🎨 視覺標準 (ECC Standard)

1.  **主要文字**：`#D4D4D8` (zinc-300) -> 這是新的「白話文」閱讀標準。
2.  **次要文字**：`#A1A1AA` (zinc-400) -> 僅限用於不重要的 metadata。
3.  **行動端行高**：至少 `1.6` 以上，讓文字「有地方呼吸」。
