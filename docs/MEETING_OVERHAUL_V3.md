## 🚨 ISSUE 1：寶物掉落沒有遊戲感

### 問題描述
用戶反映：按完「我了解了」後，寶物確實有出現，但完全沒有「遊戲掉落」的震撼感。當前只是 `opacity+scale+y` 的彈簧動畫，像 UI 元素淡入，不像寶箱從天而降爆開。

### 🏗️ Architect 提案：「寶箱墜落 → 撞擊 → 爆開」三段式動畫

```
Phase 1 (0-600ms): 寶箱本體
- 一個巨大的 🎁 emoji 或寶箱圖從螢幕頂端墜落
- 帶有 rotate 旋轉 + scale 放大
- 最後 bounce 反彈效果 (spring stiffness: 300, damping: 12)
- 螢幕震動 (screen shake)

Phase 2 (600-1200ms): 爆開
- 寶箱 scale 到 0 並消失
- 同時四散射出「光粒子」(6-8 個彩色圓形)
- 爆炸式 confetti (particleCount: 300, spread: 180)
- 出現一道向上的光柱 (emerald gradient beam)

Phase 3 (1200ms+): 內容浮現
- practice_kit 的卡片從光柱中緩慢浮現 (opacity 0→1, y: 60→0)
- 背後持續有粒子飄散
- 「UNLOCKED / 秘笈已現世」標籤帶打字機效果
```

### 🎨 UX Reviewer 補充
- 動畫必須在手機上也流暢（避免太多 DOM 元素）
- 寶箱墜落時可以加「咚！」的文字特效（文字震動）
- 整個過程不超過 2 秒，否則用戶會不耐煩

## 🚨 ISSUE 3：初次訪問新手引導系統

### 🧠 Planner 提案：「聊天式引導」取代問卷彈窗

**現狀問題**：目前的 `OnboardingModal` 是三道選擇題，太像考試，沒有溫度。

**新方案：對話式引導 (Chat-Style Onboarding)**

```
Step 1: 歡迎畫面 (全屏)
┌─────────────────────────────────┐
│                                 │
│    嗨！歡迎來到 Dee's AI Lab     │
│                                 │
│    我是 Dee，一個跟你一樣        │
│    從零開始學 AI 的普通人 🙋‍♂️   │
│                                 │
│    [🆕 我是新手，帶我逛逛]      │
│    [🎮 我有基礎，直接開始]       │
│    [📚 跳過教學，看全部文章]     │
│                                 │
└─────────────────────────────────┘

→ 選「新手」: 進入 Guided Tour（從 Lv.1 第一篇開始）
→ 選「有基礎」: 快速分級測驗（3 題簡化版）→ 定級
→ 選「跳過教學」: 直接進入完整清單模式（List View）
```

### 🏗️ Architect 資料結構

```typescript
// 新增 localStorage keys
'dee_onboarding_done': 'true' | null     // 是否完成引導
'dee_view_preference': 'guided' | 'free'  // 遊戲模式 or 自由模式
'dee_ai_level': '1' | '2' | '3'          // 當前等級
'dee_ai_completed': number[]              // 已完成文章 IDs
```

### 🎨 UX Reviewer 補充
- 「跳過教學」不能藏得太深，必須跟其他按鈕同級
- 任何時候都可以在 Insights 頁面右上角切換「遊戲模式 ↔ 自由模式」
- 自由模式 = 無鎖定、無等級、傳統 blog 列表，但仍保留 steps/quiz/treasure 的互動體驗

## 🚨 ISSUE 4：分級跳關機制

### 🧠 Planner 提案

```
模式 A：順序遊玩 (Guided Mode)
- 必須完成 Lv.1 全部 4 篇 → 自動解鎖 Lv.2
- 必須完成 Lv.2 全部 8 篇 → 自動解鎖 Lv.3
- 每篇文章有「完成狀態」(localStorage)
- 側邊顯示進度環 (4/4, 3/8, 0/4)

模式 B：跳級 (Speed Run)
- 每一級的入口有「跳級考試」按鈕
- 通過 3 題測驗 → 直接解鎖該級
- 跳級不影響個別文章的 treasure/quiz 功能

模式 C：自由模式 (Free Mode)
- 所有文章全部解鎖
- 沒有等級限制
- 進入文章後仍保留 step-by-step 互動體驗
- 但不會有 blur/lock（步驟也全部開放）
```

### 🏗️ Architect 建議
- 「自由模式」不需要移除 steps 邏輯，只需要在 ArticleDetail 裡加一個 `freeMode` flag
- `freeMode = true` 時：所有 steps 預設 `isActive`，無 blur/lock，treasure 直接可見
- 這樣只需最小改動就能同時支援兩種體驗
