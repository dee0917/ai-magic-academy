# 🏛️ ECC 團隊會議 — 教學系統 V4.0 終極體驗
**日期**：2026-02-28  
**議題**：內容完整性 + Onboarding AI 平台選擇 + 視覺遊戲感 + 慶祝特效去重

---

## 📋 用戶原始需求逐條拆解

> ① 初步學習完聊天式 AI 的底層邏輯後，就可以隨意暢遊 → **宗旨確認**
> ② 現有文章足夠讓小白理解和使用嗎？ → **內容缺口分析**
> ③ Onboarding 要問用哪個 AI、會不會用，不會的要教三大 AI → **Onboarding 重做**
> ④ 冒險/自由模式太簡陋 → **視覺大修**
> ⑤ 自由模式也要有導覽 → **導覽系統**
> ⑥ 整體沒有遊戲感，要有「哇！」的感覺 → **沉浸式 UI**
> ⑦ 教學內太多一樣的煙花，寶箱之後的慶祝要用不同效果 → **特效去重**

## 🚨 ISSUE 1：內容缺口 — 小白真的學得會嗎？

### ✍️ Content Strategist 分析

**當前 Ch.1 假設用戶已經知道：**
- 什麼是 ChatGPT / Claude / Gemini
- 怎麼開啟和註冊
- 基本的打字對話操作

**但真正的小白可能連 ChatGPT 都沒打開過。**

### 🧠 Planner 提案：新增「Chapter 0：出發前的準備」

| 順序 | 內容 | 目標 |
|------|------|------|
| 0-1 | **三大聊天 AI 是什麼？** | 介紹 ChatGPT / Claude / Gemini，各自的特色和適合誰用。附上連結和註冊教學截圖。 |
| 0-2 | **你的第一句話：跟 AI 說哈囉** | 實際操作：打開 AI → 打第一句話 → 看到回應。讓用戶建立「原來這麼簡單」的信心。 |

> **決議**：Ch.0 是「選修前導」。Onboarding 時如果用戶選「沒用過任何 AI」→ 強制進入 Ch.0。已經在用的用戶可以跳過。

### 📊 最終章節結構

```

  ├── 0-1：三大聊天 AI 是什麼？ChatGPT vs Claude vs Gemini
  └── 0-2：你的第一句話：跟 AI 說哈囉

Ch.1 🌱 認識 AI — 3 篇（不變）
Ch.2 🔧 指令進化 — 3 篇（不變）
Ch.3 🎯 生活實戰 — 4 篇（不變）
Ch.4 🏆 進階挑戰 — 2 篇（不變）

🎲 支線任務 — 6 篇（不變）
```

## 🚨 ISSUE 3：冒險/自由模式的視覺太簡陋

### 🎨 UX Reviewer 問題診斷

從截圖看到的問題：
1. **「冒險模式 / 自由模式」按鈕**：像兩個普通 tab，沒有遊戲感
2. **頁面標題區**：純文字，沒有視覺衝擊力
3. **Chapter 卡片**：跟普通 blog 列表沒差別
4. **沒有地圖感**：冒險模式應該像 RPG 地圖，而不是一個 list

### 🧠 Planner 提案：沉浸式改造

#### A. Hero 區域（頁面頂端）

```
┌──────────────────────────────────────────┐
│ ╔══════════════════════════════════════╗  │
│ ║                                    ║  │
│ ║   🧠 AI 修煉之路                   ║  │
│ ║                                    ║  │
│ ║   ████████████░░░░  58% 已探索     ║  │
│ ║   Lv.12  ·  經驗值 2,400           ║  │
│ ║                                    ║  │
│ ║   [🎮 冒險模式]  [📚 自由探索]     ║  │
│ ╚══════════════════════════════════════╝  │
│   ↑ 深色漸層卡片 + 粒子背景動畫         │
└──────────────────────────────────────────┘
```

- **經驗值系統**：每完成一篇 = +200 XP。純裝飾性的，增加成就感。
- **等級顯示**：根據完成數量計算（0=Lv.1, 3=Lv.4, 6=Lv.7...）
- **進度條**：帶 glow 的大型進度條
- **背景**：微妙的浮動粒子 + 深色漸層

#### B. 冒險模式 — Chapter 路線圖

```
Ch.0 ─── 🚀 ═══╗
                ║
Ch.1 ─── 🌱 ═══╬═══ [完成✅]
                ║
Ch.2 ─── 🔧 ═══╬═══ [進行中🔥]
                ║         ↑ 當前章節發光 + 脈動
Ch.3 ─── 🎯 ═══╬═══ [鎖定🔒] ← 模糊 + Lock
                ║
Ch.4 ─── 🏆 ═══╝

每個 Chapter 節點：
┌─────────────────────────────────────┐
│  🌱 Ch.1 · 認識 AI                 │
│  ──────────────────────             │
│  ⬤──⬤──⬤   3/3 完成              │
│  ↑ 三個圓點代表三篇文章             │
│  完成=綠色 進行中=白色 未完成=灰色   │
│                                     │
│  [展開查看文章]                      │
└─────────────────────────────────────┘
```

每個 Chapter 可以展開/收合，展開後顯示該章節的文章清單。

#### C. 自由模式 — 圖鑑式導覽

自由模式不是「無導覽」，而是像「圖鑑」或「世界地圖」：

```
┌──────────────────────────────────────────┐
│  📚 全部教學 · 18 篇                     │
│                                          │
│  ┌──── 篩選 ─────────────────────────┐   │
│  │ [全部] [入門] [技巧] [實戰] [進階] │  │
│  │ [支線] [已完成✅] [未完成]         │  │
│  └────────────────────────────────────┘   │
│                                          │
│  推薦路線：你還沒看過「打桌球互動法」      │
│  這是最多人學完後覺得最實用的一篇 →       │
│                                          │
│  ┌────┐ ┌────┐ ┌────┐                    │
│  │ 卡 │ │ 卡 │ │ 卡 │  ← 更豐富的卡片   │
│  └────┘ └────┘ └────┘                    │
└──────────────────────────────────────────┘
```

- **篩選標籤列**：可以按分類/完成狀態篩選
- **智慧推薦**：根據完成狀態，推薦「下一篇最適合你的」
- **卡片設計**：加入更多視覺元素（圖標、難度星星要更大、完成勳章）

## 🚨 ISSUE 4：慶祝特效去重

### 🔍 Code Reviewer 現狀審計

**同一篇文章中 confetti 觸發了 5 次：**

| 觸發點 | 當前效果 | 問題 |
|--------|---------|------|
| ① 複製指令 | confetti 60 發 | 太浮誇，只是複製而已 |
| ② 寶箱撞擊 | confetti 40 發 | OK（寶箱序列的一部分）|
| ③ 寶箱爆開 | confetti 600 發 (3波) | ⭐ 最精彩的一刻，保留 |
| ④ Quiz 答對 | confetti 120 發 | 跟寶箱撞感覺 |
| ⑤ 領取勳章 | confetti 200 發 | 跟寶箱撞感覺 |

### 🎨 UX Reviewer 解法：「一篇一種慶祝」原則

**每個觸發點用不同的慶祝方式，整篇文章中 confetti 只出現一次。**

| 觸發點 | 新效果 | 視覺描述 |
|--------|--------|---------|
| ① 步驟完成 | **脈動漣漪** | 按鈕點擊後，從按鈕中心擴散出一圈翡翠色光環，步驟卡變綠。安靜、乾淨。 |
| ② 複製指令 | **打勾彈跳** | 按鈕變白 + ✓ 圖標彈入 + 微微縮放。沒有煙花。 |
| ③ 寶箱掉落 | **🎁 三段式動畫** ⭐ | 保留現有的墜落→撞擊→爆開→confetti。這是唯一一次大型慶祝。 |
| ④ Quiz 答對 | **星星爆射** | 從正確選項射出 5-8 顆金色 ⭐ 向外飛散（用 framer-motion，不用 confetti）。 |
| ⑤ 領取勳章 | **金色光暈脈動** | 勳章卡片出現時，背後有金色圓環從小到大擴散 3 次，像聖光降臨。不用 confetti。 |

### 技術實作

```typescript
// 脈動漣漪（步驟完成）— 純 CSS/framer-motion
<motion.div 
    initial={{ scale: 0, opacity: 0.8 }}
    animate={{ scale: 3, opacity: 0 }}
    className="absolute inset-0 rounded-full bg-emerald-500/30"
/>

// 星星爆射（Quiz 答對）— framer-motion 動畫
const stars = Array.from({ length: 6 }, (_, i) => ({
    angle: (i / 6) * Math.PI * 2,
    distance: 80 + Math.random() * 60,
}));
{stars.map((s, i) => (
    <motion.div key={i}
        initial={{ x: 0, y: 0, opacity: 1, scale: 1 }}
        animate={{ 
            x: Math.cos(s.angle) * s.distance,
            y: Math.sin(s.angle) * s.distance,
            opacity: 0, scale: 0 
        }}
        transition={{ duration: 0.6, delay: i * 0.05 }}
        className="absolute text-xl"
    >⭐</motion.div>
))}

// 金色光暈（勳章）— 純 framer-motion
<motion.div
    animate={{ scale: [0, 2.5, 0], opacity: [0.6, 0, 0] }}
    transition={{ duration: 1.5, repeat: 2, repeatDelay: 0.3 }}
    className="absolute inset-0 rounded-full border-4 border-amber-400"
/>
```

## 🚨 ISSUE 5：Ch.0 新文章內容規劃

### ✍️ Content Strategist 規劃

#### 文章 0-1：三大聊天 AI 是什麼？

```
pain_point: "聽大家都在說 AI，但 ChatGPT、Claude、Gemini 到底差在哪？該用哪一個？"

steps:
1. "ChatGPT：最多人用的全能選手"
   - OpenAI 出品，全球用戶最多
   - 強項：寫作、程式、創意
   - 免費版就夠用，網址：chat.openai.com
   
2. "Claude：最會聽話的乖學生"  
   - Anthropic 出品，擅長長文和分析
   - 強項：邏輯推理、讀長文、遵守指令
   - 免費版就夠用，網址：claude.ai

3. "Gemini：Google 的超級大腦"
   - Google 出品，跟 Google 服務整合
   - 強項：搜尋結合、多語言、即時資訊
   - 免費版就夠用，網址：gemini.google.com

practice_kit: "三個都打開，問同一個問題，比較看看誰的回答你最喜歡！"
```

#### 文章 0-2：你的第一句話

```
pain_point: "我已經打開 AI 了，但那個空白的對話框讓我不知道要打什麼⋯⋯"

steps:
1. "第一句話：說你好"
   - 真的就打「你好」。AI 會自我介紹。冰就這樣破了。
   
2. "第二句話：問一個你已經知道答案的問題"
   - 用你已知答案的問題來「測試」AI。例如：「台北 101 有多高？」
   - 這能讓你判斷 AI 的回答品質。

3. "第三句話：請它幫你做一件小事"
   - 「幫我用 3 句話介紹一下我住的城市 [你的城市名]」
   - 恭喜！你已經完成第一次 AI 對話了 🎉

practice_kit: "複製這段話，貼進你剛剛打開的 AI 裡：「你好！我是第一次使用你。請用 3 句話告訴我，你能幫我做什麼？」"
```
